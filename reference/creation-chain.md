# Creation Chain

## Concept
A creation chain is a sequence of creating systems where each creates value
for the next. Each creating system is universal — it can serve any target.

## add-ai's Role: Methodology + Practice Layer
add-ai provides architecture-driven development capabilities:
- Full SDLC through 9 agents and 32 skills
- Architecture artifacts (ADRs, C4, kanban, iterations)
- Requirements artifacts (UC, QA, SC, CON, CRN)
- Code generation, testing, CI/CD, review

## Complementary Capabilities
add-ai agents can be enhanced by complementary capabilities if available
in the current session. Agents should describe these NEEDS, not specific
tools — the AI will match available tools/skills at runtime.

### Thinking Capabilities (upstream in chain)
Situations where first-principles thinking would help:
| Situation | Capability Need |
|-----------|----------------|
| Unclear component boundaries | Boundary analysis, decomposition review |
| Competing design alternatives | Trust-calibrated comparison (Fact/Guess/Rumor) |
| Architecture evolution planning | Explore/exploit strategy assessment |
| Terminology confusion across teams | Domain glossary with precise definitions |
| Quality metric definition | Characteristics chain (idea → metrics → scoring) |
| Architecture review | First-principles composition and boundary audit |

### Knowledge Capabilities (downstream in chain)
Situations where project knowledge retrieval would help:
| Situation | Capability Need |
|-----------|----------------|
| Reviewing inputs for ADD iteration | Prior ADRs and design decisions retrieval |
| Choosing design concepts | Existing patterns and architectural rationale |
| Understanding domain context | Domain knowledge from project sources |
| Verifying ADR compliance | Historical decision context and alternatives |

## Important
- add-ai is fully universal and works standalone
- Complementary capabilities are optional enhancements, never requirements
- Agents describe needs abstractly — never reference specific tools by name
